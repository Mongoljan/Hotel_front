## Key Interactions

### 1. Card Expansion/Collapse
- **Trigger**: Click anywhere on card header
- **Visual**: Chevron rotates (right â†’ down)
- **Animation**: Smooth expand/collapse transition
- **State**: Tracked in `expandedItems` Set

### 2. Approval Confirmation
- **Trigger**: Click any approval button
- **Dialog**: Modal overlay with confirmation
- **Actions**: Cancel (closes) or Confirm (executes)
- **Loading**: Button shows spinner during API call

### 3. Real-time Search
- **Trigger**: Typing in search box
- **Debounce**: None needed (client-side)
- **Effect**: Instant filtering of visible cards
- **Clear**: Empty search shows all

### 4. Status Updates
- **Trigger**: Successful approval/revoke
- **Effect**: 
  1. Toast notification appears
  2. Full data refresh
  3. Cards re-render with new status
  4. Statistics update

## Summary

This unified approval system provides:

1. **Single Source of Truth**: All approval data in one view
2. **Clear Status Indicators**: Color + icon + text for accessibility
3. **Flexible Actions**: Approve together or separately
4. **Efficient Navigation**: Minimal clicks, expandable details
5. **Safe Operations**: Confirmation dialogs, loading states
6. **Fast Search**: Real-time client-side filtering
7. **Responsive Design**: Works on all screen sizes

The UI prioritizes clarity, efficiency, and safety in the approval workflow while maintaining a clean, modern aesthetic consistent with the rest of the application.
